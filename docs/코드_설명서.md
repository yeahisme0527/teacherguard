### 2-1. 전역 상수 설정

#### `KST` — 한국 표준시

```python
KST = timezone(timedelta(hours=9))

def now_kst() -> datetime:
    return datetime.now(KST)
```

> Streamlit Cloud 서버는 UTC로 실행됨.  
> `datetime.now()` 를 직접 쓰면 오전 9시가 자정으로 계산되는 버그 발생.  
> → **모든 시간 처리**를 `now_kst()` 로 통일.

---

#### `ABUSE_TYPES` — 갑질 유형 정의 딕셔너리

```python
ABUSE_TYPES = {
    "욕설/폭언": {
        "keywords": ["병신", "미친", "개새", "시발", ...],
        "severity": 4,
        "emoji": "🤬",
        "color": "#D32F2F",
        "badge": "badge-red",
    },
    "협박/위협": {
        "keywords": ["고소", "죽여", "가만 안 둬", ...],
        "severity": 5,
        ...
    },
    ...총 8개 유형
}
```

- `severity` : 1~5 스케일. 4 이상이면 메시지 자동 차단.
- `keywords` : 단순 `in` 연산으로 탐지 (정규식 미사용, 속도 우선).
- `badge` : UI에서 태그 색상을 CSS 클래스로 지정.

---

#### `VARIANT_PATTERNS` — 변형어 사전

```python
VARIANT_PATTERNS = {
    "병신": ["ㅂㅅ", "뵹신", "뼝신", "병.신", "병 신"],
    "멍청이": ["멍처이", "멍청ㅇ", "멍치이", ...],
    "시발": ["ㅅㅂ", "씨발", "십8", "시팔", ...],
    ...
}
```

> 욕설을 `ㅂㅅ` 처럼 자음만 쓰거나 `병.신` 처럼 중간에 특수문자를 넣어 우회하는 시도를 탐지.  
> 변형어가 감지되면 `severity = 4` 로 고정 처리.

### 2-2. 핵심 함수들

#### `blur_text(text)` — 텍스트 블러 처리

```python
def blur_text(text: str) -> str:
    result = text
    # 1) ABUSE_TYPES 키워드 블러
    for abuse_type, data in ABUSE_TYPES.items():
        for keyword in data["keywords"]:
            if keyword in result:
                result = result.replace(keyword, "⬛" * len(keyword))
    # 2) VARIANT_PATTERNS 블러
    for word, variants in VARIANT_PATTERNS.items():
        for variant in variants:
            if variant in result:
                result = result.replace(variant, "⬛" * len(variant))
    return result
```

- 원본 텍스트(`text`)는 DB에 원본 그대로 저장 (증거용).
- 블러된 텍스트(`blurred_text`)만 화면에 표시.
- `⬛` 개수 = 원래 단어 길이 → 어느 정도 길이인지 유추 가능하게 유지.

#### `analyze_message(text, use_model=False)` — 핵심 분석 함수

```python
def analyze_message(text: str, use_model: bool = False) -> dict:
```

**처리 단계:**

```
① 빈 문자열 체크 → 기본값 반환
        ↓
② ABUSE_TYPES 키워드 순회 → detected_types, detected_keywords, max_severity 수집
        ↓
③ VARIANT_PATTERNS 변형어 순회 → 욕설/폭언 유형 추가, severity=4
        ↓
④ 강도 보정어 체크 ("진짜","정말"...) → max_severity 최대 +1
        ↓
⑤ use_model=True 이면 hate_model.classify(text) 호출
   → AI 결과를 기존 결과에 병합 (OR 방식)
        ↓
⑥ 결과 dict 반환
```

**반환 구조:**

```python
{
    "is_profanity": bool,           # severity>=4 이면 True
    "types_detected": ["욕설/폭언"], # 감지된 유형 리스트
    "detected_keywords": ["ㅂㅅ(변형)"], # 실제 감지 키워드
    "severity": 4,                  # 최고 심각도
    "emotion_level": 4,             # severity와 동일값 (감정 온도계용)
    "etiquette_warning": "...",     # 에티켓 위반 시 경고 문구
    "model_used": True,             # AI 모델 사용 여부
    "model_labels": [("악플/욕설", 0.91), ...] # AI 결과 레이블
}
```

> **`use_model` 을 전송 시에만 True로 쓰는 이유:**  
> 입력 중 실시간 분석은 키워드만 사용 (API 호출 0회).  
> 전송 버튼 클릭 시 1회만 API 호출 → 비용·속도 최적화.

```python
def get_auto_response(analysis: dict) -> str:
```

분석 결과의 `severity`, `types_detected`, 현재 시간(`is_after_hours()`)을 조합해


---

#### `is_after_hours()` — 운영시간 판단

```python
def is_after_hours() -> bool:
    now = now_kst()
    return now.hour >= 18 or now.weekday() >= 5
```

- `now.hour >= 18` : 18시 이후
- `now.weekday() >= 5` : 토(5)·일(6)
- 두 조건 중 하나라도 True면 운영시간 외

#### `init_session()` — 세션 초기화

```python
def init_session():
    defaults = {
        "auth_user": None,
        "role": None,
        "chat_messages": [],
        "teacher_room_id": None,
        "school_selected": False,
        ...
    }
    for k, v in defaults.items():

```

> Streamlit은 매 렌더(사용자 상호작용)마다 스크립트 전체를 재실행함.

#### `_hydrate_from_db()` — DB → 세션 1회 동기화

```python
def _hydrate_from_db():
    if st.session_state.get("db_loaded"):
        return  # 이미 로드됐으면 건너뜀
    ...
    # 메시지, 보관함, 차단목록, 프로필 순서로 로드
    st.session_state.db_loaded = True  # 플래그 세팅 → 중복 실행 방지
```

- `db_loaded = False` : 로그인 직후에만 False
- 로드 완료 후 `db_loaded = True` 설정
- 이후 렌더에서는 조기 반환 (DB 불필요 호출 방지)

#### `_send_parent_message(text)` — 메시지 전송 핵심 함수

```python
def _send_parent_message(text: str):
    room_id = st.session_state.get("teacher_room_id")

    # 1. AI 분석 (HuggingFace API 호출)
    if hate_model.is_enabled():
        with st.spinner("🤖 AI가 분석 중..."):
            analysis = analyze_message(text, use_model=True)
    else:
        analysis = analyze_message(text, use_model=False)

    # 2. 메시지 dict 생성
    msg = {
        "role": "parent",
        "text": text,
        "blurred_text": blur_text(text) if analysis["is_profanity"] else text,
        "blocked": analysis["emotion_level"] >= 4,
        "room_id": room_id,
        ...
    }

    # 3. session_state에 즉시 추가 (UI 반응성)
    st.session_state.chat_messages.append(msg)

    # 4. 보관함·통계 업데이트
    if analysis["is_profanity"]:
        st.session_state.archive.append(...)
        st.session_state.stats["profanity"] += 1

    # 5. AI 자동 응답 생성 & 추가
    ai_msg = {"role": "ai", "text": get_auto_response(analysis), ...}
    st.session_state.chat_messages.append(ai_msg)

    # 6. Supabase에 저장 (실패해도 앱은 정상 동작)
    if db.is_enabled():
        db.insert_message(auth_id, msg, room_id=room_id)
        db.insert_message(auth_id, ai_msg, room_id=room_id)
```

> **핵심 설계**: DB 저장 실패와 무관하게 `session_state` 는 이미 갱신됨.  
> 사용자는 오류 없이 앱을 계속 사용하고, 오류는 조용히 처리.

```python
def insert_message(user_id: str, msg: dict, room_id: str = None):
```

### 3-4. 메시지 조회 (`fetch_room_messages`)

```python
def fetch_room_messages(room_id: str, limit: int = 200):
    resp = (
        client.table("messages")
        .select("*")
        .eq("room_id", room_id)        # 특정 교사 방만
        .order("created_at", desc=True) # 최신순 내림차순
        .limit(limit)
        .execute()
    )
    rows = list(reversed(resp.data or []))  # 오래된 것이 위로
    return True, [_row_to_msg(r) for r in rows]
```

- `desc=True` 로 최신 200개 가져온 뒤 `reversed()` 로 뒤집어 시간순 정렬.
- `_row_to_msg()` 로 DB row → 앱 dict 형태 변환.

### 3-5. 초대코드 발급 (`get_or_create_invite_code`)

```python
def get_or_create_invite_code(user_id: str):
    import secrets as _sec

    # 1. 기존 코드 조회
    resp = client.table("profiles").select("invite_code").eq("id", user_id).limit(1).execute()
    existing = (resp.data or [{}])[0].get("invite_code")
    if existing:
        return True, existing

    # 2. 없으면 새 코드 생성 (충돌 재시도 최대 5회)
    chars = "ABCDEFGHJKLMNPQRSTUVWXYZ23456789"  # 헷갈리는 0·1·I·O 제외
    for _ in range(5):
        code = "".join(_sec.choice(chars) for _ in range(6))
        try:
            client.table("profiles").update({"invite_code": code}).eq("id", user_id).execute()
            return True, code
        except Exception:
            continue  # UNIQUE 충돌 시 다른 코드 재시도
```

### 3-6. `_row_to_msg()` — DB row 변환기

```python
def _row_to_msg(row: dict) -> dict:
    return {
        "role": row.get("sender_role", "parent"),
        "text": row.get("text", ""),
        "blurred_text": row.get("blurred_text") or row.get("text", ""),
        "blocked": bool(row.get("blocked", False)),
        "emotion_level": int(row.get("emotion_level", 1)),
        "timestamp": row.get("display_time") or "",
        "user_id": row.get("user_id", ""),
        "analysis": {
            "is_profanity": bool(row.get("is_profanity", False)),
            "types_detected": row.get("types_detected") or [],
            "severity": int(row.get("severity", 1)),
            "detected_keywords": row.get("detected_keywords") or [],
        },
    }
```

> DB 컬럼명과 앱 내부 dict 키 이름이 다름 (`sender_role` → `role` 등).  
> 이 함수가 두 형식 사이의 **어댑터** 역할.

## 4. hate_model.py

HuggingFace의 `kor_unsmile` 모델을 HTTP로 호출하는 래퍼.

### 4-1. API 엔드포인트 (폴백 구조)

```python
API_URLS = [
    "https://router.huggingface.co/hf-inference/models/smilegate-ai/kor_unsmile",  # 신형
    "https://api-inference.huggingface.co/models/smilegate-ai/kor_unsmile",         # 구형
]
```

두 URL을 순서대로 시도. 첫 번째 성공하면 두 번째는 호출하지 않음.

---

### 4-2. `classify(text, threshold=0.25)` 핵심 로직

```python
def classify(text: str, threshold: float = 0.25) -> dict | None:
    for url in API_URLS:
        resp = requests.post(
            url,
            headers={"Authorization": f"Bearer {token}"},
            json={"inputs": text, "options": {"wait_for_model": True}},
            timeout=25,
        )
        if resp.status_code == 200:
            labels = _parse_response(resp.json())
            if labels:
                profanity = any(l == "악플/욕설" and s >= threshold for l, s in labels)
                hate_hits = [l for l, s in labels if l in HATE_CATEGORIES and s >= threshold]
                return {
                    "profanity": profanity,
                    "hate": len(hate_hits) > 0,
                    "hate_categories": hate_hits,
                    "labels": sorted(labels, key=lambda x: -x[1]),
                }
    return None  # 두 URL 모두 실패
```

**threshold = 0.25 의 의미:**  
kor_unsmile 모델이 "악플/욕설" 레이블에 25% 이상의 확신을 보이면 탐지로 판정.  
낮을수록 민감 (false positive 증가), 높을수록 보수적 (false negative 증가).

---

### 4-3. `_parse_response(data)` — 응답 파싱

```python
def _parse_response(data) -> list[tuple[str, float]] | None:
    # [[{label,score},...]] 형식 (이중 배열)
    if isinstance(data, list) and data and isinstance(data[0], list):
        data = data[0]
    # [{label,score},...] 형식
    if isinstance(data, list):
        return [(d["label"], float(d["score"])) for d in data if "label" in d]
    return None
```

HuggingFace API가 버전에 따라 응답 형식을 다르게 줄 수 있어서 두 형식 모두 처리.

### 5-2. `_fetch_from_neis()` — 캐시 적용 API 호출

```python
@st.cache_data(ttl=86_400, show_spinner=False)  # 1일 캐시
def _fetch_from_neis(region_code: str, school_type: str) -> list[str]:
    names = []
    for page in range(1, 6):       # 페이지당 1000개, 최대 5000개
        r = requests.get(NEIS_API_URL, params={
            "KEY": key, "Type": "json",
            "pIndex": page, "pSize": 1000,
            "ATPT_OFCDC_SC_CODE": region_code,
            "SCHUL_KND_SC_NM": school_type,
        }, timeout=15)

        rows = r.json()["schoolInfo"][1].get("row", [])
        names.extend(row["SCHUL_NM"] for row in rows)
        if len(rows) < 1000:
            break  # 마지막 페이지
    return sorted(set(names))
```

## 8. Streamlit 렌더링 구조

### Streamlit의 실행 모델

```
사용자가 버튼 클릭 또는 입력
         ↓
스크립트 전체 처음부터 재실행
         ↓
session_state로 상태 복원
```

> 일반 Python 앱과 달리 Streamlit은 **이벤트 기반이 아님**.

> 따라서 상태는 반드시 `st.session_state` 에 저장해야 유지됨.
