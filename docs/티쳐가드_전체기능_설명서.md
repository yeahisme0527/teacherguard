## 1. 서비스 개요

**티쳐가드(TeacherGuard)** 는 교사를 대상으로 한 학부모 메시지 속 욕설·협박·모욕 등의 부적절한 표현을 실시간으로 감지하고 차단하는 AI 기반 교권보호 플랫폼입니다.

### 핵심 가치

| 가치 | 설명 |
|------|------|
| 🛡️ 교사 보호 | 부적절 메시지를 자동 차단해 교사의 감정 소모를 줄임 |
| 💬 건강한 소통 | 전송 전 에티켓 안내로 학부모의 자정 효과 유도 |
| 📄 증거 자동화 | 교권침해 신고서를 클릭 한 번에 자동 생성 |
| 🔒 완전한 분리 | 초대코드 기반으로 교사별 채팅방 완전 분리 |

## 2. 기술 스택

| 분류 | 기술 | 역할 |
|------|------|------|
| **프론트엔드** | Streamlit ≥ 1.32 | UI 전체 (Python 기반 웹앱) |
| **데이터베이스** | Supabase (PostgreSQL) | 메시지·프로필·차단 목록 영구 저장 |
| **인증** | Supabase Auth | 이메일/비밀번호 회원가입·로그인 |
| **AI 모델** | smilegate-ai/kor_unsmile | 한국어 혐오표현 멀티라벨 분류 |
| **AI API** | HuggingFace Inference API | kor_unsmile 모델 HTTP 호출 |
| **학교 데이터** | NEIS Open API | 전국 학교 목록 실시간 조회 |
| **배포** | Streamlit Cloud | GitHub 연동 자동 배포 |
| **언어** | Python 3.11 | 전체 백엔드 로직 |

### 주요 Python 모듈

```
streamlit_app.py  — 메인 앱 (UI + 비즈니스 로직)
db.py             — Supabase 연동 (인증·CRUD·초대코드)
hate_model.py     — HuggingFace kor_unsmile API 연동
schools.py        — NEIS 학교 검색 API 연동
```

## 3. 인증 시스템

### 3-1. 회원가입

- **이메일 + 비밀번호** 기반 가입 (Supabase Auth)
- 가입 시 `display_name` (표시 이름) 선택 입력
- 가입과 동시에 `profiles` 테이블에 프로필 자동 생성 (DB 트리거)
- 이메일 인증 정책: Supabase 대시보드에서 ON/OFF 설정 가능

### 3-2. 로그인

- `sign_in_with_password()` 호출
- 성공 시 `session_state.auth_user` 에 `user_id`, `email`, `display_name` 저장
- 로그인 직후 `_hydrate_from_db()` 1회 실행 → DB 데이터를 세션에 동기화

### 3-3. 로그아웃

- `client.auth.sign_out()` 호출
- `session_state` 전체 초기화 (auth_user, role, chat_messages, 학교정보 등)

### 3-4. 세션 유지

- Streamlit의 `st.session_state` 로 탭 내 세션 유지
- 새로고침해도 `db_loaded` 플래그로 중복 DB 호출 방지

## 5. 학교 선택 기능

교사 첫 로그인 시 자동으로 나타나는 학교 설정 화면입니다.

### 5-1. 3단계 선택 흐름

```
① 시도 선택 (드롭다운) → ② 학교급 선택 (드롭다운) → ③ 학교명 입력/선택
```

### 5-2. NEIS API 연동 모드 (키 설정 시)

- **NEIS Open API** (교육부 나이스 공공데이터)를 실시간 호출
- 지역 + 학교급 선택 → 해당 학교 목록 자동 로드
- 검색어 입력으로 필터링 (예: "서울" → 서울고, 서울중앙고, 서울외고 ...)
- `@st.cache_data(ttl=86400)` 로 결과 1일 캐싱 (API 호출 최소화)
- 지원 학교급: 유치원·초등학교·중학교·고등학교·특수학교
- 어린이집은 NEIS 미지원 → 직접 입력 모드

### 5-3. 직접 입력 모드 (키 없을 때)

- NEIS API 키 미설정 시 텍스트 입력으로 폴백
- 기능 제한 없이 동작

### 5-4. 저장 & 활용

- 저장된 학교 정보: `profiles.school_region`, `school_type`, `school_name`
- **신고서 자동 포함**: 저장한 학교명이 교권침해 신고서에 자동 반영
- **사이드바 표시**: `🏫 경남 대산고등학교` 형태로 항상 표시
- 언제든지 사이드바 → **[🏫 학교 설정]** 버튼으로 변경 가능

## 6. 초대코드 & 방 연결 시스템

교사별로 완전히 분리된 채팅방을 만드는 핵심 기능입니다.

### 6-1. 초대코드 발급 (교사)

- 교사 대시보드 최초 진입 시 **6자리 코드 자동 생성**
- 코드 형식: 대문자 + 숫자 조합 (헷갈리는 0·1·I·O 제외)  
  예: `AB3K7Q`, `XP9M2R`
- `profiles.invite_code` 컬럼에 UNIQUE 제약으로 저장
- 생성 로직: `secrets.choice()` 기반 난수 생성, 충돌 시 최대 5회 재시도
- 코드는 **고정** — 재발급하려면 DB에서 직접 삭제

### 6-2. 채팅방 입장 (학부모)

1. 6자리 코드 입력
2. `profiles` 테이블에서 `invite_code = 입력코드` 조회
3. 매칭 성공 → 교사 이름·학교 미리보기 표시
4. **[채팅방 입장]** 클릭 → `session_state.teacher_room_id` = 교사 user_id 저장
5. 해당 방의 기존 메시지 즉시 로드

### 6-3. 방(Room) 기반 메시지 분리

- 모든 메시지에 `room_id` (= 교사의 user_id) 저장
- 교사: `WHERE room_id = 본인 user_id` 로 자신의 방 메시지만 조회
- 학부모: 입장한 방의 `room_id` 로 메시지 전송
- 교사 간 메시지 노출 완전 차단

## 7. 학부모 채팅 기능

### 7-1. 메시지 작성 & 실시간 분석

- 텍스트 입력 중 **실시간 키워드 탐지** (API 호출 없이 로컬 처리)
- 탐지된 표현이 있으면 즉시 경고 배너 표시:
  ```
  ⚠️ 주의: 이 메시지는 부적절한 표현을 포함할 수 있습니다.
  🚨 이 메시지는 심각한 위협 표현을 포함합니다. 전송 시 교권보호위원회에 자동 신고될 수 있습니다.
  ```
- 탐지된 표현은 태그로 표시: `멍처이(변형)` `ㅂㅅ(변형)`

### 7-2. 전송 시 처리 흐름

```
메시지 입력
    ↓
AI 모델 분석 (kor_unsmile API) — "🤖 AI가 분석 중..."
    ↓
키워드 탐지 + AI 결과 병합
    ↓
severity 4 이상 → blocked = True
    ↓
session_state.chat_messages 에 추가
    ↓
Supabase messages 테이블에 저장 (room_id 포함)
    ↓
AI 자동 응답 메시지 생성 & 저장
```

### 7-3. 텍스트 블러 처리

- 욕설 키워드를 `⬛` 로 대체 (예: `병신` → `⬛⬛`)
- 블러된 텍스트(`blurred_text`)를 채팅창에 표시
- 원본 텍스트(`text`)는 DB에 보관 (증거용)

### 7-4. AI 자동 응답

분석 결과에 따라 아래 메시지 중 하나 자동 발송:

| 상황 | 자동 응답 예시 |
|------|--------------|
| 정상 | "✅ 메시지 잘 받았습니다. 확인 후 빠른 시일 내에 연락드리겠습니다." |
| 운영시간 외 | "🌙 현재 운영시간이 아닙니다 (18:00 이후). 내일 확인 후 답변드리겠습니다." |
| 욕설 차단 | "⚠️ 전송하신 메시지에 부적절한 표현이 포함되어 정상적으로 전달되지 않았습니다." |
| 협박 감지 | "🚨 해당 메시지는 교권보호 시스템에 의해 자동 기록되었습니다." |

## 8. 메시지 분석 엔진

`analyze_message()` 함수가 모든 분석을 담당합니다.

### 8-1. 분석 대상 유형 (ABUSE_TYPES)

| 유형 | 키워드 예시 | 심각도 |
|------|-----------|--------|
| 욕설/폭언 | 병신, 미친, 개새끼, 시발 등 | 4 |
| 협박/위협 | 고소, 죽여, 가만 안 둬 등 | 5 |
| 모욕/비하 | 무능한, 자격없는, 쓸모없는 등 | 3 |
| 차별/혐오 | 틀딱, 급식충, 한남 등 | 3 |
| 개인정보침해 | 주소 알아, 집에 찾아가 등 | 5 |
| 허위사실유포 | 아동학대, 성희롱했다 등 | 5 |
| 과도한 요구 | 당장 연락해, 즉시 처리해 등 | 2 |
| 에티켓 위반 | 야, 너, 임마 등 | 1 |

### 8-2. 변형어(VARIANT_PATTERNS) 탐지

우회 표현도 탐지합니다.

| 원형 | 변형어 예시 |
|------|-----------|
| 병신 | ㅂㅅ, 뵹신, 뼝신, 병.신 |
| 미친 | ㅁㅊ, 미칀, 미.친 |
| 시발 | ㅅㅂ, 씨발, 십8, 시팔 |
| **멍청이** | **멍처이**, 멍텅구리, 멍치이 |
| 찐따 | 찐.따, 찐다 |
| 닥쳐 | 닥처, 닥ㅊ |
| 죽어 | 뒤져, 뒤지어 |

### 8-3. 분석 결과 구조

```python
{
    "is_profanity": bool,        # 차단 여부
    "types_detected": [str],     # 감지된 유형 목록
    "detected_keywords": [str],  # 실제 감지된 키워드
    "severity": int,             # 심각도 1~5
    "emotion_level": int,        # 감정 레벨 1~5
    "model_used": bool,          # AI 모델 사용 여부
    "etiquette_warning": str     # 에티켓 경고 메시지
}
```

## 9. AI 혐오표현 탐지

### 9-1. 모델 정보

| 항목 | 내용 |
|------|------|
| 모델명 | `smilegate-ai/kor_unsmile` |
| 유형 | 한국어 혐오표현 멀티라벨 분류 |
| 분류 레이블 | 여성/가족·남성·성소수자·인종/국적·연령·지역·종교·기타혐오·악플/욕설·clean |
| 호출 방식 | HuggingFace Inference API (HTTP) — 모델 다운로드 없음 |

### 9-2. API 호출 구조

```
1차 시도: https://router.huggingface.co/hf-inference/models/smilegate-ai/kor_unsmile
2차 폴백: https://api-inference.huggingface.co/models/smilegate-ai/kor_unsmile
```

- 두 엔드포인트 모두 실패 시 → 키워드 탐지 결과만 사용 (서비스 중단 없음)
- 타임아웃: 25초
- 실패 오류: `session_state._hf_last_error` 에 기록 (디버그용)

### 9-3. 탐지 임계값 (Threshold)

- 현재값: **0.25** (낮을수록 더 민감)
- `악플/욕설` 레이블 score ≥ 0.25 → `profanity = True`
- 혐오 카테고리 레이블 score ≥ 0.25 → `hate = True`

### 9-4. 적용 시점

| 시점 | AI 사용 여부 |
|------|------------|
| 입력 중 (실시간 미리보기) | ❌ 키워드 탐지만 (API 절약) |
| **전송 버튼 클릭 시** | ✅ AI 모델 호출 |

### 9-5. 🤖 배지 표시

AI 모델로 추가 탐지된 메시지는 채팅창에 `🤖` 배지 표시

## 11. 쌍방향 채팅

### 11-1. 교사 → 학부모 답장

- **[🌡️ 실시간 모니터]** 탭 하단 답장 입력창
- `sender_role = 'teacher'` 로 저장
- `room_id = 교사 본인 user_id` 로 저장
- 학부모 화면에서 **초록색 말풍선**으로 표시

### 11-2. 채팅 메시지 역할별 표시

| 역할 | 말풍선 위치 | 색상 |
|------|-----------|------|
| 학부모 (parent) | 오른쪽 | 파란 계열 |
| AI 응답 (ai) | 왼쪽 | 회색 |
| 교사 답장 (teacher) | 왼쪽 | **초록** |

### 11-3. 새 메시지 확인

- **[🔄 새로고침]** 버튼 클릭 → `_reload_messages_from_db()` 호출
- `fetch_room_messages(room_id)` 로 최신 메시지 동기화

## 12. 보관함 & 증거 관리

### 12-1. 자동 보관 조건

아래 조건을 **하나라도** 만족하면 보관함에 자동 저장:

- `is_profanity = True` (키워드 또는 AI 탐지)
- `severity ≥ 4` (욕설·협박·개인정보침해 등)

### 12-2. 보관함 정보

각 메시지에 대해 표시:
- 전송 시간
- 원본 메시지 (블러 처리된 버전)
- 감지된 유형 태그 (예: `욕설/폭언` `협박/위협`)
- 심각도 (Severity 1~5)
- **[🚫 차단]** / **[📤 신고서 포함]** 버튼

### 12-3. 데이터 영속성

- Supabase `messages` 테이블의 `is_profanity = true` 행
- 로그인 시 자동 복원 (새로고침해도 유지)

## 16. 운영시간 필터

### 16-1. 설정

- 운영시간: **평일 07:00 ~ 18:00 KST**
- Streamlit Cloud는 UTC 서버 → 모든 시간 판단을 **KST(UTC+9)** 기준으로 처리

### 16-2. 시간외 처리

- 메시지 전송 자체는 가능 (차단 없음)
- AI 자동 응답: `"🌙 현재 운영시간이 아닙니다..."` 로 대체
- 채팅창 상태 표시: `🌙 운영시간 외 (18:00 이후)`

### 16-3. KST 처리 코드

```python
KST = timezone(timedelta(hours=9))

def now_kst() -> datetime:
    return datetime.now(KST)
```

## 17. 데이터베이스 연동

### 17-1. Supabase 테이블 구조

#### `profiles` — 사용자 프로필
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | auth.users 와 1:1 연결 |
| email | text | 이메일 |
| display_name | text | 표시 이름 |
| school_region | text | 시도 (예: 경남) |
| school_type | text | 학교급 (예: 고등학교) |
| school_name | text | 학교명 (예: 대산고등학교) |
| invite_code | text UNIQUE | 6자리 초대코드 |

#### `messages` — 채팅 메시지
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | 메시지 고유 ID |
| user_id | uuid | 발신자 user_id |
| room_id | uuid | 교사 user_id (방 식별자) |
| sender_role | text | parent / ai / teacher |
| text | text | 원본 텍스트 |
| blurred_text | text | 블러 처리된 텍스트 |
| blocked | boolean | 차단 여부 |
| is_profanity | boolean | 부적절 메시지 여부 |
| emotion_level | int | 감정 레벨 1~5 |
| severity | int | 심각도 1~5 |
| types_detected | jsonb | 감지된 유형 배열 |
| detected_keywords | jsonb | 감지된 키워드 배열 |
| display_time | text | HH:MM 표시용 |

#### `blocked_users` — 차단 목록
| 컬럼 | 타입 | 설명 |
|------|------|------|
| teacher_id | uuid | 차단한 교사 |
| blocked_user_id | text | 차단된 학부모 ID |

### 17-2. Row Level Security (RLS)

| 테이블 | 정책 |
|--------|------|
| profiles | 인증 사용자 전체 조회, 본인만 수정 |
| messages | 본인 room_id 조회, 본인 user_id로만 insert |
| blocked_users | 본인(teacher_id) 것만 조회·수정·삭제 |

### 17-3. 폴백 동작

Supabase 미설정 시 → **게스트 모드** (session_state 기반, 새로고침 시 초기화)
